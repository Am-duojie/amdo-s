# IDE 中合并分支的多种方法

## 🎯 直接回答

**不一定要用命令！** 在 IDE 中有多种方法可以合并分支，包括纯图形界面的方法。

---

## 📋 方法对比

### 方法1：使用命令面板（最常用）

```
Ctrl+Shift+P → Git: Merge Branch...
```

**优点**：
- ✅ 快速
- ✅ 明确

**缺点**：
- ⚠️ 需要输入命令

---

### 方法2：使用源代码管理面板菜单（纯图形界面）

#### 步骤：

1. **打开源代码管理面板**
   - 点击左侧边栏的 **Git 图标**（或按 `Ctrl+Shift+G`）

2. **打开菜单**
   - 在源代码管理面板顶部，点击 **"..."** 菜单（三个点）

3. **选择合并**
   - 选择 **"分支"** → **"合并分支..."**
   - 或直接选择 **"合并分支..."**（如果有）

4. **选择要合并的分支**
   - 在弹出的列表中选择：`experiment/ai-recommendation`
   - 点击或按 Enter

✅ **完全不需要输入命令！**

---

### 方法3：使用图形视图（如果支持）

某些 Git 扩展（如 Git Graph）提供图形界面：

1. **打开图形视图**
   - 在源代码管理面板找到"图形"区域
   - 或安装 Git Graph 扩展

2. **右键点击分支**
   - 在图形视图中找到要合并的分支
   - 右键点击 → 选择"合并到当前分支"

✅ **纯图形操作！**

---

### 方法4：使用终端（命令行）

虽然你问的是图形界面，但也可以：

```
git merge experiment/ai-recommendation
```

## 🎨 详细步骤：纯图形界面方法

### 使用源代码管理面板菜单

#### 完整步骤：

```
1. 点击左侧边栏的 Git 图标（或按 Ctrl+Shift+G）

   ✅ 如果有冲突，会提示解决冲突
```

## 💡 实用技巧

### 技巧1：查看可合并的分支

在合并前，可以先查看有哪些分支：

```
1. 点击源代码管理面板的 "..." 菜单
2. 选择 "分支" → "查看所有分支"
3. 可以看到所有本地和远程分支
```

### 技巧2：快速切换分支

合并前需要切换到主分支：

```
方法1（图形界面）：
- 点击底部状态栏的分支名
- 选择 "main"

方法2（菜单）：
- 源代码管理面板 → "..." 菜单
- 分支 → 检出到...
- 选择 "main"
```

### 技巧3：查看合并结果

合并后，可以在图形视图中查看：

```
1. 在源代码管理面板的"图形"区域
2. 可以看到合并提交
3. 可以看到分支的合并点
```

## ⚠️ 注意事项

### 1. 合并前确保在主分支

```
✅ 正确：
1. 先切换到 main 分支
2. 然后合并

❌ 错误：
1. 在实验分支上合并
2. 会合并到错误的分支
```

### 2. 合并前更新主分支

```
✅ 正确：
1. 切换到 main
2. 点击"同步更改"（拉取最新代码）
3. 然后合并

❌ 错误：
1. 直接合并
2. 可能覆盖其他人的提交
```

### 3. 处理合并冲突

如果出现冲突：

```
1. 在"更改"区域会显示冲突文件
2. 点击文件，查看冲突内容
3. 手动解决冲突
4. 暂存并提交
```
