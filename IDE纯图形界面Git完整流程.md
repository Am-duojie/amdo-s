## 1. 创建实验分支

### 方法A：使用底部状态栏（最简单）

#### 步骤：

```
1. 查看底部状态栏（左下角）
   会显示当前分支名，例如：[main]

2. 点击分支名 [main]

3. 在弹出的菜单中，选择：
   "创建新分支..." 或 "Create new branch..."

4. 输入分支名：
   experiment/ai-recommendation

5. 按 Enter 确认

✅ 已创建并切换到新分支！
底部状态栏会显示：[experiment/ai-recommendation]
```

### 方法B：使用源代码管理面板

#### 步骤：

```
1. 点击左侧边栏的 Git 图标（或按 Ctrl+Shift+G）
   ✅ 打开源代码管理面板

2. 在面板顶部，点击 "..." 菜单（三个点）

3. 选择 "分支" → "创建分支..."

4. 输入分支名：
   experiment/ai-recommendation

5. 按 Enter 确认

✅ 已创建新分支！
```

## 2. 在实验分支上开发

### 步骤：

```
1. 确认当前分支
   - 查看底部状态栏，应该显示：[experiment/ai-recommendation]
   - 如果显示 [main]，说明还在主分支，需要先切换

2. 正常编辑代码
   - 在 IDE 中打开文件
   - 进行修改
   - 保存文件（Ctrl+S）

3. 修改的文件会自动显示
   - 在左侧"源代码管理"面板的"更改"区域
   - 会显示所有修改的文件
```

### 查看修改内容：

```
1. 在"更改"区域，点击文件名
2. IDE 会显示：
   - 左侧：修改前的代码
   - 右侧：修改后的代码
   - 中间：对比标记
```

## 3. 提交代码

### 完整步骤：

#### 步骤1：暂存文件（添加到暂存区）

**方法A：暂存单个文件**
```
1. 在"更改"区域，找到要提交的文件
2. 点击文件旁边的 "+" 号
   ✅ 文件已添加到暂存区
   文件会移动到"暂存的更改"区域
```

**方法B：暂存所有文件**
```
1. 在"更改"区域顶部，点击 "全部暂存" 按钮
   ✅ 所有修改的文件都已添加到暂存区
```

#### 步骤2：输入提交信息

```
1. 在"消息"输入框中输入提交信息
   例如："实验：添加AI推荐功能"

2. 提交信息要清晰描述做了什么
   ✅ 好的提交信息：
   - "实验：添加AI推荐功能"
   - "修复支付回调bug"
   - "更新首页样式"
   
   ❌ 不好的提交信息：
   - "更新"
   - "修改"
   - "提交"
```

#### 步骤3：提交代码

**方法A：使用提交按钮**
```
1. 点击"提交"按钮（✓ 图标）
   ✅ 代码已提交到当前分支
```

**方法B：使用快捷键**
```
1. 在"消息"输入框中
2. 按 Ctrl+Enter
   ✅ 代码已提交
```

#### 步骤4：确认提交成功

```
1. 提交后，"暂存的更改"区域会清空
2. "更改"区域也会清空（如果没有其他修改）
3. 在"图形"区域可以看到新的提交
```

## 4. 切换回主分支

### 步骤：

```
1. 点击底部状态栏的分支名
   当前显示：[experiment/ai-recommendation]

2. 在弹出的菜单中，选择 "main"

3. ✅ 已切换到 main 分支
   底部状态栏显示：[main]
```

### 注意事项：

```
⚠️ 如果有未提交的修改：
- IDE 会提示有未保存的更改
- 可以选择：
  1. 先提交修改（推荐）
  2. 暂存修改（git stash）
  3. 放弃修改（不推荐）
```

## 5. 合并实验分支

### 完整步骤：

#### 步骤1：确保在主分支

```
1. 查看底部状态栏
2. 应该显示：[main]
3. 如果不是，先切换到 main 分支
```

#### 步骤2：更新主分支（重要！）

```
1. 点击底部状态栏的"同步更改"图标（两个箭头）
   或者
2. 在源代码管理面板，点击 "..." 菜单
   选择 "拉取" 或 "Pull"

✅ 确保主分支是最新的
```

#### 步骤3：合并实验分支

**方法A：使用源代码管理面板菜单（推荐）**

```
1. 点击左侧 Git 图标（打开源代码管理面板）

2. 在面板顶部，点击 "..." 菜单（三个点）

3. 选择 "分支" → "合并分支..."

4. 在弹出的列表中选择要合并的分支：
   experiment/ai-recommendation

5. 点击或按 Enter

✅ 合并完成！
```

**方法B：使用命令面板**

```
1. 按 Ctrl+Shift+P 打开命令面板

2. 输入：Git: Merge Branch...

3. 按 Enter

4. 选择要合并的分支：
   experiment/ai-recommendation

5. 按 Enter

✅ 合并完成！
```

#### 步骤4：确认合并成功

```
1. 在"图形"区域可以看到合并提交
2. 实验分支的代码现在在 main 分支了
3. 如果有冲突，会显示在"更改"区域
```

## 6. 推送代码到GitHub

### 步骤：

#### 方法A：使用同步更改按钮（推荐）

```
1. 点击底部状态栏的"同步更改"图标（两个箭头）
   ✅ 自动推送代码到 GitHub
```

#### 方法B：使用源代码管理面板

```
1. 点击左侧 Git 图标

2. 在面板顶部，点击 "..." 菜单

3. 选择 "推送" 或 "Push"

✅ 代码已推送到 GitHub
```

#### 方法C：使用命令面板

```
1. 按 Ctrl+Shift+P

2. 输入：Git: Push

3. 按 Enter

✅ 代码已推送
```

### 确认推送成功：

```
1. 推送后，底部状态栏的"同步更改"图标会消失
2. 在"图形"区域可以看到远程标记（origin/main）
3. 可以在 GitHub 网站上查看最新代码
```

## 7. 删除实验分支

### 步骤：

#### 删除本地分支：

```
1. 确保不在要删除的分支上
   - 切换到 main 分支

2. 按 Ctrl+Shift+P 打开命令面板

3. 输入：Git: Delete Branch...

4. 选择要删除的分支：
   experiment/ai-recommendation

5. 按 Enter

✅ 本地分支已删除
```

#### 删除远程分支（如果推送过）：

```
1. 按 Ctrl+Shift+P

2. 输入：Git: Delete Remote Branch...

3. 选择：origin/experiment/ai-recommendation

4. 按 Enter

✅ 远程分支已删除
```


## 9. 处理冲突

### 识别冲突：

```
合并时如果出现冲突：
1. 在"更改"区域会显示冲突文件
2. 文件旁边会有红色标记 ⚠️
3. 底部状态栏会显示冲突数量
```

### 解决冲突：

#### 步骤1：打开冲突文件

```
1. 在"更改"区域，点击冲突文件
2. IDE 会显示冲突内容
```

#### 步骤2：查看冲突标记

```
冲突文件会显示：

<<<<<<< HEAD
主分支的代码
=======
实验分支的代码
>>>>>>> experiment/ai-recommendation
```

#### 步骤3：解决冲突

**方法A：手动编辑**
```
1. 删除冲突标记（<<<<<<< ======= >>>>>>>）
2. 保留你想要的代码
   - 保留主分支的代码
   - 保留实验分支的代码
   - 或者合并两者
3. 保存文件（Ctrl+S）
```

**方法B：使用IDE的冲突解决工具**
```
1. 在冲突标记上方，会显示按钮：
   - "接受当前更改"（主分支）
   - "接受传入的更改"（实验分支）
   - "接受两者"
   - "比较更改"
2. 点击相应的按钮
3. IDE 会自动解决冲突
```

#### 步骤4：标记冲突已解决

```
1. 解决冲突后，文件会显示在"更改"区域
2. 点击文件旁的 "+" 添加到暂存区
3. 输入提交信息："解决合并冲突"
4. 点击"提交"按钮
✅ 冲突已解决并提交
```

## 10. 日常开发流程

### 场景1：开发新功能（使用分支）

```
1. 创建实验分支
   点击底部状态栏 → 创建新分支... → experiment/新功能

2. 在实验分支上开发
   - 编辑代码
   - 在"更改"区域暂存文件（点击 "+"）
   - 输入提交信息
   - 点击"提交"按钮

3. 功能完成后合并
   - 切换到 main 分支
   - 点击"同步更改"更新主分支
   - 源代码管理面板 → "..." 菜单 → 分支 → 合并分支...
   - 选择 experiment/新功能

4. 推送代码
   - 点击"同步更改"图标

5. 删除实验分支（可选）
   - Ctrl+Shift+P → Git: Delete Branch...
```

### 场景2：日常提交（在主分支）

```
1. 编辑代码

2. 暂存文件
   - 点击文件旁的 "+"
   - 或点击"全部暂存"

3. 输入提交信息
   - 在"消息"框输入描述

4. 提交代码
   - 点击"提交"按钮
   - 或按 Ctrl+Enter

5. 推送代码
   - 点击"同步更改"图标
```

### 场景3：获取最新代码

```
1. 点击底部状态栏的"同步更改"图标
   ✅ 自动拉取并合并远程代码

2. 如果有冲突，按照"处理冲突"步骤解决
```

## ⚠️ 注意事项

### 1. 提交前检查

```
✅ 应该做的：
- 查看"更改"区域，确认要提交的文件
- 查看文件差异，确认修改正确
- 输入清晰的提交信息

❌ 不应该做的：
- 提交临时文件
- 提交敏感信息（密码、密钥等）
- 提交无法运行的代码
```

### 2. 合并前准备

```
✅ 应该做的：
- 切换到主分支
- 更新主分支（拉取最新代码）
- 确保实验分支已提交

❌ 不应该做的：
- 在主分支上直接实验
- 不更新就合并
- 有未提交的修改就合并
```

### 3. 推送前确认

```
✅ 应该做的：
- 确认代码可以运行
- 确认没有敏感信息
- 确认提交信息清晰

❌ 不应该做的：
- 推送无法运行的代码
- 推送包含密码的代码
```
