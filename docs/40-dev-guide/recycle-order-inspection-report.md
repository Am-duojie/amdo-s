# 回收订单质检报告编辑器

## 概述

回收订单的质检模块已升级为和官方验商品编辑中一样的66项检测编辑器。

## 访问方式

1. 登录管理后台
2. 进入 **回收订单管理**
3. 点击订单的 **质检** 按钮（状态为"已寄出"或"已收货"时显示）
4. 在弹出的质检报告对话框中编辑66项检测内容

## 功能特性

### 1. 66项详细检测

- **外观检测**（12项）：碎裂、划痕、机身弯曲等
- **屏幕检测**（16项）：触控、外观、显示
- **设备功能**（30项）：按键、生物识别、传感器、接口、无线、充电、通话、声音、摄像头等
- **维修浸液**（8项）：屏幕、主板、机身、零件等

### 2. 可折叠界面

- 默认收起状态，节省空间
- 支持展开/收起每个分类
- 一键全部展开/收起

### 3. 异常标记

- 每个检测项可以标记为"正常"或"异常"
- 异常项可以添加异常图片URL
- 异常项可以填写详细描述

### 4. 价格设置

- 根据质检结果设置最终价格
- 支持额外加价
- 支持质检备注

## 使用流程

### 1. 开始质检

点击订单的"质检"按钮，打开质检报告对话框。

### 2. 编辑检测项

1. 点击分类标题展开检测项
2. 修改检测结果（如"无"、"正常"、"几乎不可见"）
3. 如有异常，选择"异常"并填写异常描述和图片URL

### 3. 设置价格

1. 根据质检结果设置最终价格
2. 如需加价，填写加价金额
3. 填写质检备注（可选）

### 4. 保存

点击"保存并完成质检"按钮，系统会：
- 保存66项检测数据
- 更新最终价格
- 将订单状态更新为"已检测"

## 数据格式

质检数据以 JSON 格式存储在 `check_items` 字段中：

```json
[
  {
    "title": "外观检测",
    "images": [],
    "groups": [
      {
        "name": "外壳外观",
        "items": [
          {
            "label": "碎裂",
            "value": "无",
            "pass": true
          },
          {
            "label": "划痕",
            "value": "明显划痕",
            "pass": false,
            "image": "https://example.com/scratch.jpg"
          }
        ]
      }
    ]
  }
]
```

## 与官方验商品的区别

### 相同点

- 66项检测内容完全相同
- 界面和交互方式相同
- 数据结构相同

### 不同点

- 回收订单质检后会自动更新订单状态为"已检测"
- 回收订单需要设置最终价格
- 回收订单支持加价功能

## 注意事项

1. **必填项**：最终价格是必填项
2. **异常图片**：异常图片需要先上传到服务器，然后填写URL
3. **价格设置**：最终价格应根据质检结果合理设置
4. **状态更新**：保存后订单状态会自动更新为"已检测"
