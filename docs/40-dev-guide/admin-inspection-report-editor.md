# 管理后台质检报告编辑器使用指南

## 概述

管理后台现在提供了可视化的质检报告编辑器，可以直接在编辑商品时编辑66项检测内容。

## 访问编辑器

1. 登录管理后台
2. 进入 **官方验商品管理**
3. 点击任意商品的 **编辑** 按钮
4. 在编辑表单中找到 **详细质检报告（66项检测）** 卡片

## 使用方法

### 1. 展开/收起分类

点击右上角的 **全部展开/全部收起** 按钮可以快速展开或收起所有检测分类。

### 2. 编辑检测项

每个检测项包含以下字段：

- **检测项名称**：固定的检测项目名称（如"碎裂"、"划痕"等）
- **检测结果**：输入框，填写检测结果（如"无"、"正常"、"几乎不可见"等）
- **状态**：下拉选择，选择"正常"或"异常"
- **异常图片URL**：仅当状态为"异常"时显示，填写异常图片的URL

### 3. 标记异常项

如果某个检测项有问题：

1. 将状态改为 **异常**
2. 在检测结果中描述问题（如"明显划痕"、"检测到坏点"）
3. 填写异常图片URL（可选）

### 4. 保存数据

编辑完成后，点击底部的 **保存** 或 **保存并上架** 按钮。

质检报告数据会自动保存到 `inspection_reports` 字段中。

## 4大检测分类

### 1. 外观检测（12项）

- 碎裂
- 划痕
- 机身弯曲
- 脱胶/缝隙
- 外壳/其他
- 磕碰
- 刻字/图
- 掉漆/磨损
- 摄像头/闪光灯外观
- 褶皱
- 卡托
- 音频网罩

### 2. 屏幕检测（16项）

**屏幕触控（1项）**
- 触控

**屏幕外观（7项）**
- 屏幕/其它
- 碎裂
- 内屏掉漆/划伤
- 屏幕凸点（褶皱）
- 支架破损
- 浅划痕
- 深划痕

**屏幕显示（8项）**
- 进灰
- 坏点
- 气泡
- 色斑
- 其它
- 亮点/亮斑
- 泛红/泛黄
- 图文残影

### 3. 设备功能（30项）

包含10个分组：
- 按键（4项）
- 生物识别（2项）
- 传感器（4项）
- 接口（2项）
- 无线（4项）
- 充电（2项）
- 通话功能（2项）
- 声音与振动（3项）
- 摄像头（3项）
- 其它状况（1项）

### 4. 维修浸液（8项）

- 屏幕
- 主板
- 机身
- 零件维修/更换
- 零件缺失
- 后摄维修情况
- 前摄维修情况
- 浸液痕迹情况

## 常见检测结果示例

### 正常状态
- 碎裂：无
- 划痕：无
- 触控：正常
- 电源键：正常
- WiFi：正常

### 轻微问题
- 磕碰：几乎不可见
- 浅划痕：几乎不可见
- 掉漆/磨损：几乎不可见

### 异常状态
- 划痕：明显划痕（状态选择"异常"）
- 坏点：检测到坏点（状态选择"异常"）
- 屏幕：检测到维修更换（状态选择"异常"）

## 前端展示

编辑保存后，用户在商品详情页可以看到：

- 正常项：显示为黑色文字，带绿色✓图标
- 异常项：显示为红色文字，带"查看异常图"按钮

## 数据格式

质检报告数据以 JSON 格式存储在 `inspection_reports` 字段中：

```json
[
  {
    "title": "外观检测",
    "images": [],
    "groups": [
      {
        "name": "外壳外观",
        "items": [
          {
            "label": "碎裂",
            "value": "无",
            "pass": true
          },
          {
            "label": "划痕",
            "value": "明显划痕",
            "pass": false,
            "image": "https://example.com/scratch.jpg"
          }
        ]
      }
    ]
  }
]
```

## 注意事项

1. **异常图片**：异常图片需要先上传到服务器，然后填写URL
2. **数据验证**：保存时会自动验证数据格式
3. **默认值**：新建商品时，所有检测项默认为"正常"状态
4. **批量编辑**：暂不支持批量编辑质检报告，需要逐个商品编辑

## 快捷操作

- **全部展开**：快速查看所有检测项
- **全部收起**：收起所有分类，节省空间
- **快速标记异常**：将状态改为"异常"即可
