## 第一部分：项目概述（3-4页）

### 第1页：封面
- 项目名称：二手交易与回收综合平台
- 学生姓名、学号、专业
- 指导教师

### 第2页：项目背景与意义
- **背景**：
  - 二手交易市场快速发展
  - 电子设备回收需求增长
  - 用户对质量保障的需求
- **意义**：
  - 提供便捷的二手交易平台
  - 支持设备回收变现

### 第3页：项目定位
- **核心定位**："二手交易 + 回收 + 官方验"综合平台

  - 二手交易（C2C）

- **双端架构**：用户端 + 管理端

### 第4页：项目目标
- **功能目标**：实现完整的二手交易与回收业务流程

- **质量目标**：代码规范、文档完善、可维护性强

## 第二部分：技术架构（4-5页）

### 第5页：技术栈选型
- **后端**：
  - Django 5.2.8 + DRF（REST API）
  - SimpleJWT（认证）
  - Django Channels（WebSocket）
  - MySQL 8.0（数据库）
- **前端**：
  - Vue 3 + Composition API
  - Vite 5.x（构建工具）
  - Element Plus（UI组件库）
  - Pinia（状态管理）
- **第三方**：支付宝（支付集成）

### 第6页：系统架构图
```
客户端 (Vue 3)
    ↓ HTTP/WebSocket
后端 API 层
  ├─ 用户端业务 (secondhand_app)
  ├─ 管理端业务 (admin_api)
  ├─ 支付模块 (payment_views)
  └─ WebSocket (Channels)
    ↓
数据层 (MySQL)
```
- **架构特点**：前后端分离、模块化设计、RESTful API

### 第7页：模块划分
- **后端模块**：
  - `secondhand_app/`：用户端业务（商品、订单、回收、私信等）
  - `admin_api/`：管理端业务（订单管理、统计、权限等）
  - `consumers.py`：WebSocket实时通信
- **前端模块**：
  - `pages/`：用户端页面
  - `admin/pages/`：管理端页面
  - `components/`：公共组件
  - `stores/`：状态管理

### 第8页：数据库设计
- **核心模型**：
  - 用户端：User、Product、Order、RecycleOrder等
  - 官方验：VerifiedDevice、VerifiedProduct、VerifiedOrder
  - 管理端：AdminUser、AdminInspectionReport
- **设计特点**：状态机模式、外键关联、时间戳字段

## 第三部分：核心功能（6-7页）

### 第9页：用户端功能概览
- **账号与认证**：注册/登录、个人信息、头像上传
- **商品模块**：发布/编辑、搜索/筛选、详情展示
- **交易订单**：下单、支付、状态跟踪
- **社交功能**：收藏、私信（WebSocket实时通信）
- **回收业务**：估价问卷、订单提交、进度跟踪
- **官方验**：商品浏览、下单、订单跟踪

### 第10页：二手交易流程
- **流程图**：
  ```
  浏览商品 → 查看详情 → 私信沟通 → 下单 → 支付 → 发货 → 确认收货 → 完成
  ```
- **订单状态机**：pending → paid → shipped → completed
- **关键功能**：商品搜索、多图片上传、在线支付

### 第11页：回收业务流程
- **流程图**：
  ```
  提交订单 → 填写物流 → 平台收货 → 质检评估 → 价格确认 → 打款 → 完成
  ```
- **状态机**：
  - 流程状态：pending → quoted → confirmed → shipped → inspected → completed
  - 打款状态：pending → paid（与流程状态分离）
- **关键功能**：估价问卷、质检报告（66项检测）、价格确认、打款

### 第12页：官方验业务流程
- **流程图**：
  ```
  回收订单质检通过 → 创建库存设备 → 发布为官方验商品 → 用户下单 → 订单处理 → 完成
  ```
- **设备状态**：pending → repairing → ready → listed → sold
- **商品状态**：draft → pending → active → sold
- **关键功能**：设备库存管理、商品审核、质检报告展示

### 第13页：管理端功能概览
- **仪表盘与统计**：数据统计、回收履约效率分析、SLA超时率、价格差距分布
- **回收订单管理**：列表筛选、详情查看、物流管理、质检报告编辑、定价、打款
- **官方验管理**：设备库存管理、商品管理、审核队列
- **基础数据管理**：分类、商品、用户、消息等

### 第14页：支付集成
- **支付宝集成**：
  - 支付创建与跳转
  - 支付回调验签（RSA2）
  - 支付状态查询
  - 转账功能（打款）
  - 结算功能
- **关键特性**：回调幂等性、状态一致性、日志留痕

### 第15页：实时通信
- **WebSocket实现**：Django Channels
- **功能**：私信消息实时推送、已读状态同步
- **架构**：ASGI模式，支持异步处理

## 第四部分：关键技术实现（3-4页）

### 第16页：认证与授权
- **JWT Token认证**：无状态认证，支持刷新机制
- **权限控制**：基于角色的权限控制（RBAC），细粒度权限点
- **安全措施**：Token过期机制、权限校验

### 第17页：支付安全
- **回调验签**：RSA2签名验证，防止伪造回调
- **幂等性保证**：防止重复处理导致数据不一致
- **状态一致性**：订单状态与支付状态同步
- **日志留痕**：关键操作记录日志，便于对账与审计

### 第18页：状态机设计
- **订单状态机**：严格的状态流转规则
- **回收订单状态机**：流程状态与打款状态分离
- **设备状态机**：库存设备状态管理
- **优势**：保证数据一致性、防止非法状态流转

### 第19页：数据统计与分析
- **统计指标**：
  - 回收业务：订单量、完成率、平均价格
  - 二手交易：订单量、销售额
  - 履约效率：SLA超时率、各环节耗时
- **数据导出**：CSV格式导出
- **可视化**：仪表盘图表展示
