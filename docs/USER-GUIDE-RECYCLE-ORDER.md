# 回收订单用户操作指南

## 完整流程概览

```
估价问卷 → 估价详情 → 提交订单 → 填写物流 → 寄出设备 → 确认价格 → 等待打款
```

## 详细操作步骤

### 第一步：完成估价问卷

1. 访问回收估价页面
2. 选择设备类型（手机、平板、笔记本等）
3. 选择品牌（苹果、华为、小米等）
4. 选择型号（iPhone 13、华为 Mate 60 等）
5. 选择存储容量（128GB、256GB 等）
6. 选择成色（全新、几乎全新、良好、一般、较差）
7. 回答设备状况问题（可选）
8. 点击"下一步"或"查看估价"

**系统操作**：
- 自动计算预估价格
- 跳转到估价详情页

---

### 第二步：查看估价详情

**页面显示**：
- 页面标题："估价详情"
- 设备信息：品牌 / 型号 / 存储容量
- 预估价格：大字显示，例如 ¥3,200
- 价格明细：
  - 基础价格：¥3,500
  - 成色调整：-¥300 (91%)
  - 额外加价：+¥50（如有活动）
- 邮寄方式：平台收件地址
- 收款信息：支付宝账号或手机号
- 费用概览：快递费用说明

**用户操作**：
1. 仔细查看预估价格和明细
2. 确认收款信息是否正确
3. 如需修改收款信息，点击"修改收款信息"
4. 确认无误后，点击"提交订单"按钮

**确认对话框**：
- 提示："确认提交订单吗？提交后需要填写物流信息并寄出设备。"
- 按钮："确认提交" / "再想想"

---

### 第三步：提交订单

**点击"确认提交"后**：
1. 系统创建订单，状态为"待寄出"
2. 显示成功提示："订单提交成功！请填写物流信息"
3. 自动跳转到订单详情页

**订单详情页显示**：
- 订单号：#123
- 订单状态：待寄出（蓝色标签）
- 设备信息：品牌、型号、存储容量、成色
- 预估价格：¥3,200
- 联系信息：联系人、电话、地址
- 操作提示：蓝色信息框 "订单已提交，请填写物流信息并寄出设备"
- 操作按钮："填写物流信息"（蓝色大按钮）

---

### 第四步：填写物流信息

**点击"填写物流信息"按钮**：

弹出对话框，标题："填写物流信息"

**表单内容**：
1. 物流公司（必填）：
   - 下拉选择：顺丰速运、圆通速递、申通快递、中通快递、韵达快递、EMS、其他
2. 运单号（必填）：
   - 文本输入框
   - 提示："请输入运单号"

**操作**：
1. 选择物流公司
2. 填写运单号
3. 点击"确认"按钮

**提交后**：
1. 显示成功提示："物流信息已提交！订单状态已更新为已寄出"
2. 对话框关闭
3. 订单状态自动变为"已寄出"
4. 页面刷新，显示物流信息

**页面更新后显示**：
- 订单状态：已寄出（绿色标签）
- 物流信息卡片：
  - 物流公司：顺丰速运
  - 运单号：SF1234567890
  - 寄出时间：2025/12/14 15:30:00
- 操作提示：蓝色信息框 "设备已寄出，等待平台收货"

---

### 第五步：寄出设备

**用户操作**：
1. 妥善包装设备
2. 按照填写的物流信息寄出设备
3. 将设备寄到平台收件地址：
   - 收件人：TESTV回收 15608348253
   - 地址：重庆市九龙坡区经纬大道1099号附78号

**注意事项**：
- 建议使用顺丰或其他可靠快递
- 包装要牢固，避免运输损坏
- 保留快递单据，以便查询物流
- 可以在订单详情页查看物流信息

**等待平台收货**：
- 订单状态保持"已寄出"
- 可以随时查看订单详情
- 平台收到设备后会确认收货

---

### 第六步：平台收货和质检

**管理员操作**（用户无需操作）：

1. **确认收货**：
   - 平台收到设备后，管理员点击"确认收到设备"
   - 订单状态变为"已收货"
   - 用户可以在订单详情页看到收货时间

2. **质检设备**：
   - 管理员进行66项质检
   - 检测外观、屏幕、功能、维修记录等
   - 拍摄异常照片（如有）
   - 填写质检备注
   - 设置最终价格和加价（如有）
   - 保存质检报告

3. **质检完成**：
   - 订单状态变为"已检测"
   - 用户收到通知（可选）

**用户看到的变化**：
- 订单状态：已检测（绿色标签）
- 新增质检报告卡片
- 显示最终价格和加价
- 显示操作提示和按钮

---

### 第七步：确认最终价格

**订单详情页显示**：

**蓝色信息框**：
- 标题："质检已完成，请确认最终价格"
- 内容：
  - 最终价格：¥3,000（红色大字）
  - 加价：+¥50（绿色）
  - 实付金额：¥3,050（橙色大字）

**质检报告卡片**：
- 质检时间：2025/12/14 18:00:00
- 质检项目：66项检测结果
- 异常项目：用红色标注
- 质检备注：管理员填写的说明

**操作按钮**：
1. "确认最终价格"（蓝色大按钮）
2. "对最终价格有异议"（黄色按钮）

#### 情况A：确认价格

**点击"确认最终价格"**：

弹出确认对话框：
- 标题："确认最终价格"
- 内容："确认接受最终价格 ¥3,000 + 加价 ¥50 = ¥3,050 吗？确认后订单将进入打款阶段。"
- 按钮："确认" / "取消"

**点击"确认"后**：
1. 显示成功提示："已确认最终价格，订单已进入打款阶段"
2. 订单状态变为"已完成"
3. 页面刷新

**页面更新后显示**：
- 订单状态：已完成（绿色标签）
- 绿色成功框："订单已完成，等待平台打款中..."
- 不再显示操作按钮

#### 情况B：提出价格异议

**点击"对最终价格有异议"**：

弹出对话框：
- 标题："最终价格异议"
- 表单：
  - 异议原因（必填）：文本域，4行
  - 提示："请说明您对最终价格的异议原因"
- 按钮："提交异议" / "取消"

**填写异议原因并提交**：
1. 显示成功提示："异议已提交，等待管理员处理"
2. 订单状态保持"已检测"
3. 页面刷新

**页面更新后显示**：
- 订单状态：已检测（绿色标签）
- 黄色警告标签："价格异议"
- 异议原因显示在页面上
- 等待管理员重新评估

**管理员处理异议后**：
- 管理员查看异议原因
- 重新评估并调整价格
- 保存新的最终价格
- 用户需要重新确认价格

---

### 第八步：等待打款

**订单状态：已完成**

**页面显示**：
- 订单状态：已完成（绿色标签）
- 绿色成功框："订单已完成，等待平台打款中..."
- 最终价格：¥3,000
- 加价：¥50
- 实付金额：¥3,050
- 打款状态：待打款

**管理员打款后**：
1. 管理员在后台点击"打款给用户"
2. 选择打款方式：
   - 支付宝转账：直接转账到用户支付宝
   - 钱包：充值到平台钱包
3. 确认打款

**打款完成后**：
- 打款状态：已打款（绿色标签）
- 打款信息卡片：
  - 打款方式：支付宝转账
  - 打款账户：user@alipay.com
  - 打款时间：2025/12/14 20:00:00
  - 打款备注：回收订单#123打款

**用户查看余额**：
- 如果是支付宝转账：直接到账支付宝
- 如果是钱包：进入"我的钱包"查看余额

## 订单状态说明

| 状态 | 显示文字 | 标签颜色 | 说明 |
|------|---------|---------|------|
| pending | 待寄出 | 蓝色 | 订单已创建，等待用户填写物流信息 |
| shipped | 已寄出 | 绿色 | 用户已填写物流信息并寄出设备 |
| received | 已收货 | 绿色 | 平台已收到设备，等待质检 |
| inspected | 已检测 | 绿色 | 质检完成，等待用户确认价格 |
| completed | 已完成 | 绿色 | 用户已确认价格，等待打款 |
| cancelled | 已取消 | 灰色 | 订单已取消 |

## 常见问题

### Q1: 提交订单后可以修改物流信息吗？
A: 在"待寄出"状态下，可以重新填写物流信息。一旦状态变为"已寄出"，就无法修改。

### Q2: 如果对最终价格不满意怎么办？
A: 可以点击"对最终价格有异议"按钮，填写异议原因。管理员会重新评估并调整价格。

### Q3: 确认价格后可以反悔吗？
A: 确认后无法撤销。如果有问题，请联系客服处理。

### Q4: 打款需要多久？
A: 通常在用户确认价格后1-3个工作日内完成打款。

### Q5: 如何查看打款记录？
A: 进入"我的钱包"页面，可以查看所有交易记录。

### Q6: 设备寄出后多久能收到？
A: 通常2-5个工作日，具体取决于物流速度和距离。

### Q7: 质检需要多久？
A: 平台收到设备后，通常1-2个工作日完成质检。

### Q8: 可以取消订单吗？
A: 在"待寄出"和"已寄出"状态下可以取消订单。一旦进入质检阶段，无法取消。

## 注意事项

1. **设备包装**：请妥善包装设备，避免运输损坏
2. **物流选择**：建议使用顺丰等可靠快递
3. **保留单据**：保留快递单据，以便查询物流
4. **及时确认**：质检完成后请及时确认价格，避免延误打款
5. **收款信息**：确保收款信息准确无误
6. **价格异议**：如有异议，请详细说明原因，便于管理员处理
