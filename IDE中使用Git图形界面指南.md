# 在 IDE 中使用 Git 图形界面 - 完整指南

## 🎯 你看到的界面说明

从你的 IDE 界面来看，这是 **VS Code** 的源代码管理面板，包含：

- **更改 (Changes)** 区域：显示未提交的文件
- **图形 (Graph)** 区域：显示提交历史
- **提交按钮**：可以直接提交代码

---

## 📋 在 IDE 中实现实验性功能开发

### 方法1：使用底部状态栏（最简单）

#### 步骤1：创建实验分支

1. **查看当前分支**
   - 看 IDE 底部状态栏（左下角）
   - 会显示：`main` 或 `main*`（*表示有未提交的修改）

2. **点击分支名称**
   - 点击状态栏上的 `main`
   - 会弹出分支选择菜单

3. **创建新分支**
   - 在弹出菜单中，点击 **"创建新分支..."** 或 **"Create new branch..."**
   - 输入分支名：`experiment/ai-recommendation`
   - 按 Enter 确认
   - ✅ 自动创建并切换到新分支

**可视化步骤**：
```
底部状态栏: [main] 
    ↓ 点击
弹出菜单: 
  - main
  - 创建新分支...
    ↓ 点击，输入 experiment/ai-recommendation
✅ 已切换到新分支
```

#### 步骤2：在实验分支上开发

1. **确认当前分支**
   - 看底部状态栏，应该显示：`experiment/ai-recommendation`

2. **正常编辑代码**
   - 在 IDE 中编辑文件
   - 修改会显示在"更改"区域

3. **提交代码**
   - 在"更改"区域，你会看到修改的文件
   - 点击文件旁边的 **"+"** 号，添加到暂存区
   - 或者点击 **"全部暂存"** 按钮
   - 在"消息"输入框输入提交信息：`实验：添加AI推荐功能`
   - 点击 **"提交"** 按钮（或按 `Ctrl+Enter`）

#### 步骤3A：实验成功，合并到主分支

1. **切换回主分支**
   - 点击底部状态栏的分支名：`experiment/ai-recommendation`
   - 选择 `main` 分支
   - ✅ 自动切换

2. **合并实验分支**
   - 按 `Ctrl+Shift+P` 打开命令面板
   - 输入：`Git: Merge Branch`
   - 选择要合并的分支：`experiment/ai-recommendation`
   - ✅ 自动合并

3. **推送到 GitHub**
   - 点击底部状态栏的 **"同步更改"** 图标（两个箭头）
   - 或按 `Ctrl+Shift+P`，输入：`Git: Push`

4. **删除实验分支**（可选）
   - 按 `Ctrl+Shift+P`
   - 输入：`Git: Delete Branch`
   - 选择：`experiment/ai-recommendation`

#### 步骤3B：实验失败，放弃实验

1. **切换回主分支**
   - 点击底部状态栏，选择 `main`

2. **删除实验分支**
   - 按 `Ctrl+Shift+P`
   - 输入：`Git: Delete Branch`
   - 选择：`experiment/ai-recommendation`
   - ✅ 实验代码完全删除，主分支不受影响

### 方法2：使用源代码管理面板

#### 创建分支

1. **打开源代码管理面板**
   - 点击左侧边栏的 **Git 图标**（或按 `Ctrl+Shift+G`）

2. **查看分支**
   - 在源代码管理面板顶部，点击 **"..."** 菜单
   - 选择 **"分支"** → **"创建分支..."**
   - 输入分支名：`experiment/ai-recommendation`
   - 按 Enter

3. **切换分支**
   - 在源代码管理面板顶部，点击当前分支名
   - 选择要切换的分支

#### 提交代码

1. **暂存文件**
   - 在"更改"区域，每个文件旁边有 **"+"** 号
   - 点击 **"+"** 添加到暂存区
   - 或点击 **"全部暂存"** 按钮

2. **输入提交信息**
   - 在"消息"输入框输入：`实验：添加AI推荐功能`

3. **提交**
   - 点击 **"提交"** 按钮
   - 或按 `Ctrl+Enter`

4. **推送**
   - 提交后，点击 **"同步更改"** 按钮（两个箭头）
   - 或点击 **"..."** 菜单 → **"推送"**

---

### 方法3：使用命令面板（最灵活）

#### 所有操作都可以通过命令面板完成

1. **打开命令面板**
   - 按 `Ctrl+Shift+P`（Windows/Linux）
   - 或按 `Cmd+Shift+P`（Mac）

2. **常用命令**
   ```
   Git: Create Branch...          # 创建分支
   Git: Checkout to...            # 切换分支
   Git: Merge Branch...           # 合并分支
   Git: Delete Branch...          # 删除分支
   Git: Push                      # 推送到远程
   Git: Pull                      # 从远程拉取
   Git: Commit                    # 提交
   Git: Stage All Changes         # 暂存所有更改
   ```

## 💡 实用技巧

### 技巧1：查看当前分支
- 看底部状态栏左侧，显示当前分支名
- 如果有未提交的修改，会显示 `*` 号

### 技巧2：快速暂存所有文件
- 在"更改"区域，点击 **"全部暂存"** 按钮
- 或按 `Ctrl+Shift+P` → `Git: Stage All Changes`

### 技巧3：查看文件差异
- 在"更改"区域，点击文件名
- 会显示修改前后的对比

### 技巧4：撤销暂存
- 已暂存的文件会显示在"暂存的更改"区域
- 点击文件旁的 **"-"** 号可以取消暂存

### 技巧5：查看提交历史
- 在"图形"区域可以看到所有提交
- 点击提交可以查看详情

## ⚠️ 注意事项

### 1. 切换分支前确保已提交
- 如果有未提交的修改，切换分支可能会失败
- 建议先提交或暂存当前工作

### 2. 合并前确保主分支是最新的
- 切换回 main 分支后
- 点击"同步更改"拉取最新代码
- 然后再合并实验分支

### 3. 分支命名规范
- 使用清晰的分支名：`experiment/功能名`
- 避免使用空格和特殊字符
