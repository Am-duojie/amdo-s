## 📁 方法一：文件历史查看（最常用）

这是最直接、最常用的方法，适合查看单个文件的完整历史。

### 步骤详解

#### 1. 访问文件
```
访问路径：https://github.com/Am-duojie/----
→ 进入文件目录（如 backend/）
→ 点击文件（如 估价算法说明.md）
```

#### 2. 打开历史记录
在文件页面，你会看到右上角有几个按钮：
- **Raw** - 查看原始文件
- **Blame** - 查看逐行修改记录
- **History** ⏰ - **查看完整历史**（点击这个！）

#### 3. 浏览历史版本
点击 "History" 后，你会看到：
```
📅 提交时间 | 👤 提交者 | 📝 提交信息
├─ 2024-01-15 - 张三 - 更新估价算法说明
├─ 2024-01-10 - 张三 - 添加价格计算示例
└─ 2024-01-05 - 张三 - 初始文档创建
```

#### 4. 查看特定版本
- 点击任意一个提交记录（例如：点击 "更新估价算法说明"）
- 页面会跳转到该提交的详情页
- 可以看到：
  - ✅ 这个提交修改了哪些文件
  - ✅ 每个文件的修改内容（绿色=新增，红色=删除）
  - ✅ 完整的文件内容（点击文件名查看）

### 实际示例

假设你想查看 `backend/估价算法说明.md` 的历史：

1. **打开仓库**：`https://github.com/Am-duojie/----`
2. **进入文件**：点击 `backend/` → 点击 `估价算法说明.md`
3. **查看历史**：点击右上角 "History" 按钮
4. **选择版本**：点击你想查看的提交记录
5. **查看内容**：在提交详情页，点击文件名查看完整内容

## 📊 方法二：查看所有提交

适合查看整个项目的提交历史，了解项目的发展过程。

### 步骤详解

#### 1. 访问提交页面
有两种方式：

**方式 A：从仓库主页**
```
访问：https://github.com/Am-duojie/----
→ 点击顶部导航栏的 "commits" 链接
```

**方式 B：直接访问 URL**
```
https://github.com/Am-duojie/----/commits/main
```

#### 2. 浏览提交列表
你会看到按时间倒序排列的所有提交：
```
📝 提交信息                    👤 作者    📅 时间         📁 文件数量
├─ 修复登录验证bug            张三      2小时前       1个文件修改
├─ 添加商品搜索功能           张三      1天前        3个文件修改
├─ 更新API文档               张三      3天前        2个文件修改
└─ ...
```

#### 3. 查看提交详情
- 点击提交信息（如 "修复登录验证bug"）
- 可以看到：
  - 📋 提交信息详情
  - 📁 修改的文件列表
  - ➕➖ 每个文件的具体修改（新增/删除的行数）
  - 🔍 点击文件名查看完整修改内容

#### 4. 查看提交时的文件内容
在提交详情页：
- 点击任意文件名（如 `backend/app/views.py`）
- 会显示该文件在这个提交时的完整内容
- 可以看到该提交对文件的修改

### 实用功能

✅ **搜索提交**
- 在提交页面顶部有搜索框
- 可以输入关键词搜索提交信息
- 例如：搜索 "登录" 可以找到所有与登录相关的提交

✅ **按作者筛选**
- 在提交页面，可以点击作者名
- 查看该作者的所有提交

✅ **按日期筛选**
- GitHub 会自动按日期分组
- 可以看到 "Today"、"This week"、"Earlier" 等分组

## 🔍 方法三：Blame 视图

Blame 视图可以查看文件中每一行代码的修改历史和作者信息。

### 什么是 Blame？

Blame（追责视图）显示：
- 每一行代码是谁写的
- 什么时候写的
- 在哪个提交中写的

### 如何使用

#### 1. 打开 Blame 视图
```
访问文件 → 点击右上角 "Blame" 按钮
```

#### 2. 查看逐行信息
每行代码左边会显示：
```
[提交哈希] [提交信息] [作者] [时间]
```

#### 3. 查看行的历史
- 点击行号左侧的提交哈希
- 会显示该行在该提交中的内容
- 可以看到该行是如何被修改的

### 实际应用场景

✅ **查找问题代码的作者**
- 发现一个 bug，想知道是谁写的
- 使用 Blame 快速定位

✅ **了解代码演进**
- 查看某段代码的修改历史
- 理解代码为什么这样写

✅ **代码审查**
- 审查代码时，了解每行的来源
- 便于提问和讨论

## 💾 方法四：下载历史版本

如果需要下载某个历史版本的文件到本地。

### 方法 A：从文件历史下载

1. **打开文件历史**（参考方法一）
2. **选择版本**：点击你想下载的提交
3. **查看文件**：在提交详情页，点击文件名
4. **下载文件**：
   - 点击 "Raw" 按钮查看原始内容
   - 右键 → "另存为" 下载
   - 或直接复制内容

### 方法 B：通过提交哈希访问

如果你知道提交哈希（commit hash）：

```
格式：
https://github.com/用户名/仓库名/blob/提交哈希/文件路径

示例：
https://github.com/Am-duojie/----/blob/abc123def/backend/估价算法说明.md
```

### 方法 C：克隆特定提交

如果需要下载整个项目在某个提交时的状态：

```bash
# 克隆仓库
git clone https://github.com/Am-duojie/----.git

# 切换到特定提交
git checkout <提交哈希>
```

## 🔄 方法五：对比不同版本

GitHub 可以对比两个版本之间的差异，非常实用。

### 对比两个提交

#### 1. 访问提交页面
```
https://github.com/Am-duojie/----/commits/main
```

#### 2. 选择对比
- 找到两个你想对比的提交
- 点击提交信息左侧的复选框
- 点击 "Compare" 按钮

#### 3. 查看差异
会显示两个版本之间的所有差异：
- 🟢 **绿色**：新增的内容
- 🔴 **红色**：删除的内容
- 修改的部分会同时显示删除和新增

### 对比文件的两个版本

#### 方法 1：在文件历史中对比
1. 打开文件历史
2. 选择两个提交
3. GitHub 会自动显示差异

#### 方法 2：使用 Compare 功能
在文件页面：
- 点击文件路径上方的分支/提交下拉菜单
- 选择两个不同的提交或分支
- 自动显示差异

### 实际应用

✅ **查看功能改动**
- 对比功能上线前后的代码
- 了解具体改了什么

✅ **代码审查**
- 对比修改前后的代码
- 审查代码质量

✅ **问题排查**
- 对比出问题前后的版本
- 找出问题原因

在提交详情页：
- 可以看到修改统计：`+50 -20`（新增50行，删除20行）

## 🎯 常见问题

### Q1: 找不到 History 按钮？
- 确保你在文件详情页，不是在文件夹列表
- 刷新页面重试
- 检查文件是否在 Git 管理下（是否有提交记录）

### Q2: 历史记录太多，如何查找？
- 使用提交页面的搜索功能
- 按作者筛选
- 按时间范围查找（GitHub 会按日期分组）

### Q3: 可以恢复到历史版本吗？
- 在 GitHub 网页上不能直接恢复
- 需要在本地 Git 操作：
  ```bash
  git checkout <提交哈希> -- <文件路径>
  git commit -m "恢复文件到历史版本"
  git push
  ```

### Q4: 如何查看被删除的文件历史？
- 在提交历史中找到删除该文件的提交
- 点击该提交
- 可以看到删除前的文件内容
